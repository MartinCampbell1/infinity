Infinity Control Plane Topologies

Static architecture diagrams for local development, staging delivery, and the production target. These diagrams document topology and trust boundaries; readiness still comes from validation and release gates.

Local Developer Topology

Loopback-only shell, work-ui, kernel, local state, and optional migration database.

Staging Delivery Topology

Hosted shell/work-ui, private control plane services, GitHub/Vercel delivery proof, and object storage.

Production Target Topology

Target state for production readiness. This is not a claim that the current branch is production-ready.

Local

  • Loopback defaults for shell, work-ui, and kernel.
  • File-backed state is acceptable for local development.
  • Full validation may start owned services and browser automation.

Staging

  • Requires explicit hosted origins and credentials.
  • External delivery smoke proves PR, preview, CI, and artifacts.
  • Local file paths are rejected from hosted proof.

Production Target

  • Durable Postgres, service auth, audit, and worker isolation.
  • Signed immutable artifacts and observable runtime state.
  • No production wording without production-tier proof.
